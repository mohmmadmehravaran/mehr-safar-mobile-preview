# مهر سفر — رزرو هتل (نسخه با ویرایشگر بصری زنده)

پلتفرم جستجو و رزرو آنلاین هتل با یک **ویرایشگر بصری زنده شبیه فیگما** که با
آن می‌توانید صفر تا صد ظاهر سایت را با کلیک تغییر دهید.

## امکانات ویرایشگر بصری

- **ویرایش زنده با کلیک:** روی هر عنصر (متن، تصویر، دکمه، کادر و ...) کلیک کنید
  تا انتخاب شود و از پنل کناری (Inspector) متن، رنگ، فونت، حاشیه، سایه و... را
  تغییر دهید.
- **جابجایی و تغییر اندازه:** عناصر را با دستگیره سبز بکشید و با ۸ نقطه اطراف،
  اندازه‌شان را تغییر دهید.
- **خروج / ورود سریع به حالت ویرایش:**
  - دکمه **«خروج»** در نوار بالا حالت ویرایش را می‌بندد تا مثل یک بازدیدکننده
    عادی بین صفحات بگردید.
  - دکمه شناور **«ویرایش بصری»** (پایین-چپ، فقط برای مدیر) دوباره ویرایش را
    فعال می‌کند — از هر صفحه‌ای.
  - منوی **«صفحات»** در نوار بالا برای پرش بین صفحات بدون خروج از حالت ویرایش.
- **مگنت / اسنپ (Snap):** هنگام جابجایی و تغییر اندازه، عناصر به‌صورت آهنربایی
  به لبه‌ها و مرکز عناصر مجاور و وسط صفحه می‌چسبند و خطوط راهنمای صورتی نمایش
  داده می‌شوند. با دکمه **«مگنت روشن/خاموش»** قابل کنترل است.
- **نمایش فاصله‌ها (Spacing):** با انتخاب هر عنصر، فاصلهٔ آن تا کادر والد از
  چهار طرف (چپ، راست، بالا، پایین) با خط‌چین و عدد پیکسلی نمایش داده می‌شود.

> ورود مدیر: مسیر `/#/admin` با نام کاربری `admin` و رمز `admin123`.

## اجرا به‌صورت محلی

```bash
npm install
npm run dev
```

## ساخت نسخهٔ تولید

```bash
npm run build      # خروجی تک‌فایلی در dist/index.html
```

## استقرار (Deploy)

با هر push روی شاخهٔ `main`، گردش‌کار GitHub Actions (`.github/workflows/deploy.yml`)
پروژه را ساخته و روی **GitHub Pages** منتشر می‌کند.

## 🃏 جدید: سازندهٔ کارت زندهٔ داخل صفحه (مثل «کارها» در پنل مدیریت)

حالا هنگام ساخت/ویرایش هر صفحهٔ سفارشی در «ویرایش بصری»، می‌توانید دقیقاً
با همان تنظیمات بخش «کارها» در پنل مدیریت، روی همان صفحه کارت بسازید و
به‌صورت زنده نتیجه را ببینید:

- در نوار ابزار ویرایش بصری دکمهٔ جدید **«🃏 کارت‌ها»** اضافه شده است.
- با زدن آن، یک پنل شناور و قابل‌جابجایی باز می‌شود که **همان فرم کامل ساخت کارت**
  (بخش‌ها، چیدمان رو/زیر هم، اندازهٔ کارت، نوع هتل/شهر/بنر، عنوان، تصویر، لینک و…)
  را در خود دارد — منتها **مخصوص همان صفحه**.
- هر تغییری که در پنل بدهید، **بلافاصله و زنده** روی همان صفحه نمایش داده می‌شود.
- وقتی از «📄 صفحات → ساختن» یک صفحهٔ جدید می‌سازید، خودکار به آن صفحه می‌روید و
  پنل «کارت‌ها» باز می‌شود تا فوراً شروع به افزودن کارت کنید.
- کارت‌ها به‌صورت **مجزا برای هر صفحه** ذخیره می‌شوند؛ کارت‌های صفحهٔ اصلی و
  پنل مدیریت مثل قبل کار می‌کنند (سازگاری کامل با نسخهٔ قبل).

## تغییرات نسخه جدید — ویرایشگر بصری حرفه‌ای (فتوشاپ + کانوا)

قابلیت‌های اضافه‌شده به «ویرایش بصری»:

### مدیریت لایه‌ها (مثل فتوشاپ)
- **پنل لایه‌ها** شناور و قابل‌جابجایی (دکمه «لایه‌ها» در نوار ابزار): فهرست همه عناصر صفحه به ترتیب رو‌هم‌قرارگرفتن.
- **کشیدن‌و‌رها (Drag & Drop)** برای تغییر ترتیب لایه‌ها (رو هم / زیر هم).
- **نمایش/مخفی کردن** هر لایه با آیکون چشم 👁.
- **قفل/بازکردن قفل** لایه 🔒 (لایه قفل‌شده جابجا یا تغییر اندازه نمی‌شود).
- **تغییر نام** لایه (دابل‌کلیک یا آیکون مداد).
- **کپی** و **حذف** لایه از داخل پنل.
- دکمه‌های «یک لایه جلوتر/عقب‌تر» و «کاملاً جلو/عقب».

### امکانات طراحی (مثل کانوا)
- **کتابخانه فونت** فارسی و لاتین (وزیرمتن، لاله‌زار، امیری، نسخ، کوفی، Poppins، Playfair و…) با بارگذاری خودکار از Google Fonts.
- **چرخش عنصر (Rotation)** با اسلایدر و پیش‌فرض‌های ۰/۴۵/۹۰ درجه.
- **ترازبندی در صفحه**: چپ‌چین، وسط‌چین، راست‌چین نسبت به عرض صفحه.
- **کپی سریع** عنصر (Duplicate).

### میانبرهای کیبورد
- جهت‌نماها: جابجایی ۱ پیکسلی (با Shift: ۱۰ پیکسل)
- `Delete` / `Backspace`: حذف عنصر
- `Ctrl/Cmd + D`: کپی عنصر
- `Ctrl/Cmd + ]` و `Ctrl/Cmd + [`: جلو/عقب بردن لایه
- `L`: قفل/بازکردن قفل لایه
- `Esc`: لغو انتخاب

## 🛠️ بک‌اند (API)

این پروژه اکنون یک **بک‌اند کامل و حرفه‌ای** دارد که در پوشه‌ی [`server/`](./server) قرار گرفته است.

- **تکنولوژی:** Node.js + Express + TypeScript + Prisma (پیش‌فرض SQLite، قابل تغییر به PostgreSQL)
- **احراز هویت:** JWT برای کاربر و مدیر، رمزها با bcrypt هش می‌شوند
- **اعتبارسنجی:** همه‌ی ورودی‌ها با Zod
- **امنیت:** helmet، CORS، محدودسازی نرخ درخواست (rate limiting)
- **داده‌ها:** همان ۱۲ هتل، ۲۵ اتاق، رزروها و نظرات به‌صورت خودکار seed می‌شوند
- **موتور رزرو:** بررسی ظرفیت و تداخل تاریخ، کد پیگیری عمومی برای هر رزرو
- **آماده‌ی استقرار:** Dockerfile، `render.yaml`، و GitHub Actions CI

### اجرای سریع بک‌اند

```bash
cd server
cp .env.example .env      # سپس JWT_SECRET و رمز مدیر را تغییر دهید
npm install
npx prisma generate
npx prisma db push
npm run db:seed
npm run dev               # http://localhost:4000
```

مستندات کامل API و فهرست endpointها در [`server/README.md`](./server/README.md).

> توجه: GitHub Pages فقط فایل‌های استاتیک (فرانت‌اند) را میزبانی می‌کند. بک‌اند را روی
> یک سرویس Node مانند **Render** (فایل `render.yaml` آماده است) یا با **Docker** مستقر کنید.

## 🚀 استقرار یک‌مرحله‌ای (فرانت + بک‌اند با یک لینک)

این پروژه به‌صورت **یک سرویس واحد** قابل استقرار است: بک‌اند هم API را سرو می‌کند
و هم فرانت‌اند build‌شده را. کافیست روی Render یک Blueprint بسازید:

1. وارد [Render](https://dashboard.render.com/blueprints) شوید → **New → Blueprint**.
2. ریپوی `mehr-safar-booking` را انتخاب کنید (فایل `render.yaml` در ریشه به‌صورت خودکار خوانده می‌شود).
3. **Apply** را بزنید. Render فرانت را build می‌کند، در `server/public` کپی می‌کند، بک‌اند را build و دیتابیس را seed می‌کند و سرویس را بالا می‌آورد.
4. پس از چند دقیقه یک آدرس مانند `https://mehr-safar-app.onrender.com` می‌گیرید که **کل اپ** روی آن کار می‌کند.

فرانت‌اند به‌صورت پیش‌فرض API را از همان دامنه (`/api`) می‌خواند، پس نیازی به تنظیم اضافه نیست.
برای استقرار جدا (فرانت روی Pages و بک‌اند جای دیگر) متغیر `VITE_API_BASE` را هنگام build فرانت تنظیم کنید.

> دیتابیس پیش‌فرض SQLite است؛ روی پلن رایگان Render دیسک موقت است و با هر ری‌استارت
> داده‌ها دوباره seed می‌شوند (برای تست عالی است). برای ماندگاری، یک دیسک Render اضافه کنید
> یا provider را در `prisma/schema.prisma` به PostgreSQL تغییر دهید.
